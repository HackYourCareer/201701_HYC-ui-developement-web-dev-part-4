Promises

Promise to proxy dla wartości, która będzie dostępna w momencie, gdy wywołanie zostanie skonczone, a sama wartość uzupełniona. Wtedy mowimy o spełnieniu promise'a.
A promise is a proxy for a value not necessarily known at its creation time.
A promise is a proxy for a value not necessarily known at its creation time.

Promises give us a way to handle asynchronous processing in a more synchronous fashion. 
function dodajChmiel(chmiel, function() {
  function schładzajBrzeczke(temperatura, function() {
    callback();
  });
});

let dodajChmiel = new Promise(resolve => {
    resolve('dodajChmiel');
  }),
  schładzajBrzeczke = new Promise(resolve => {
    resolve('schładzajBrzeczke');
  });

dodajChmiel
  .then(schładzajBrzeczke)
  .catch(
    // some code
  )
Jak widac po tym przykładzie zwracany promise jest obiektem, który udostępnia nam metodę then(fn: onResolve, fn: onReject), pozwalającą obsłużyć promise w momencie, gdy zostaje on już obsłużony.

modern js - destructuring, kompozycja, Property Shorthand

the new way


function Programmer(characteristics) {
  let {name, surname, langSupport} = characteristics;

  let code = function() {
    console.log(`my name is ${name} and i\'m coding in ${langSupport}`);
  }

  let introduceSelf = function() {
    console.log(`i\'m ${name} ${surname}`)
  }

  return Object.freeze({
    code,
    introduceSelf
  });
}
